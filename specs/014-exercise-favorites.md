# Specification: Избранные упражнения

**Epic:** E2 — Тренировки
**User Story:** Пользователь хочет помечать упражнения как избранные, чтобы быстро находить «свои» при выборе в тренировку — без листания всего каталога и без повторного поиска.
**Related specs:** [004-exercise-catalog.md](004-exercise-catalog.md), [005-workout-tracking.md](005-workout-tracking.md)

---

## 1. User Goal

Пользователь видит каталог упражнений и хочет, чтобы 10–15 «своих» (которые он реально делает) всегда были под рукой: на отдельной вкладке и в верху списка внутри своей категории.

---

## 2. Context

Избранные — это user-scoped поверх глобального каталога (`exercises`). Никаких пользовательских упражнений (spec 004 §2) — только метка «нравится». Удаление упражнения из каталога каскадно убирает его из избранного.

---

## 3. User Scenarios

### Scenario 1 — Пометить упражнение как избранное

**How to test independently**: Открыть пикер упражнений, тапнуть звёздочку на тайле любого упражнения, убедиться, что иконка становится залитой и при повторном открытии состояние сохраняется.

**Acceptance criteria**:

1. **Given** пользователь на экране выбора упражнения.
   **When** он тапает звёздочку на тайле.
   **Then** упражнение добавляется в избранное, иконка показывает заполненное состояние.

2. **Given** упражнение уже в избранном.
   **When** пользователь тапает звёздочку.
   **Then** упражнение удаляется из избранного, иконка показывает контурное состояние.

3. **Given** один и тот же пользователь дважды добавляет одно и то же упражнение (race / двойной тап).
   **Then** дубликата в `user_exercise_favorites` нет (PK по `(user_id, exercise_id)`); ответ API одинаков.

### Scenario 2 — Отдельная вкладка «Избранные»

**How to test independently**: Открыть пикер, переключиться на вкладку «Избранные», увидеть только помеченные упражнения.

**Acceptance criteria**:

1. **Given** пользователь добавил ≥1 упражнение в избранное.
   **When** он открывает вкладку «Избранные» в пикере.
   **Then** список содержит ровно эти упражнения, отсортированные по дате добавления (новые сверху).

2. **Given** избранных нет.
   **When** пользователь открывает вкладку «Избранные».
   **Then** показывается empty state «Пока пусто. Помечайте упражнения звёздочкой на вкладке "Все"».

### Scenario 3 — Избранные в верху категорий

**How to test independently**: Помечаем 2 упражнения в категории «Грудь», открываем фильтр «Грудь» на вкладке «Все», проверяем, что они в начале.

**Acceptance criteria**:

1. **Given** пользователь на вкладке «Все», выбрана категория.
   **When** список загружается.
   **Then** избранные упражнения текущей категории идут первыми (sort `is_favorite DESC, exercise_name ASC`).

## 4. Data Model

Таблица `user_exercise_favorites`:

| column        | type                       | notes                                       |
|---------------|----------------------------|---------------------------------------------|
| `user_id`     | uuid, FK users.id CASCADE  | PK part 1                                   |
| `exercise_id` | uuid, FK exercises.id CASCADE | PK part 2                                |
| `created_at`  | timestamptz, default now() | используется для сортировки на вкладке «Избранные» |

Композитный PK `(user_id, exercise_id)` обеспечивает идемпотентность POST.

---

## 5. API

| method  | path                                  | назначение                                  |
|---------|---------------------------------------|---------------------------------------------|
| GET     | `/exercises`                          | возвращает `is_favorite` для каждого item, сортировка favorites-first |
| GET     | `/exercises/favorites`                | список избранных текущего пользователя      |
| POST    | `/exercises/{exercise_id}/favorite`   | пометить (идемпотентно) — 204               |
| DELETE  | `/exercises/{exercise_id}/favorite`   | снять метку (идемпотентно) — 204            |

`POST/DELETE` отвечают `204 No Content` (нет тела, операция идемпотентная). Несуществующее `exercise_id` → 404.

---

## 6. Non-Functional

- N+1 запрещён: `is_favorite` считается одним LEFT JOIN в `GET /exercises`.
- Каскадное удаление при удалении пользователя или упражнения.
- На юзера разумно ограничить (~500 избранных), но в MVP не реализуем — соблюдаем YAGNI.

# Specification: Аналитика и отчёты

**Epic:** E6 — Аналитика и отчёты
**User Story:** Пользователь должен видеть графики прогресса (вес, % жира, мышечная масса, тоннаж тренировок), сравнивать InBody-замеры между собой и иметь возможность экспортировать всё это в PDF.
**Related specs:** [003-inbody-measurements.md](003-inbody-measurements.md), [005-workout-tracking.md](005-workout-tracking.md), [008-ai-inbody-predictor.md](008-ai-inbody-predictor.md)

---

## 1. User Goal

Пользователь хочет видеть на одном экране, как меняется его тело и работа в зале. График веса, % жира, мышечной массы, объём проделанной работы — всё в одном месте. Когда у него есть встреча с врачом или тренером, он хочет экспортировать отчёт в PDF и отдать его.

---

## 2. Context

Аналитика — это представление поверх существующих данных:
- InBody-измерения ([003](003-inbody-measurements.md)) → графики тела
- Workouts + ExerciseLogs ([005](005-workout-tracking.md)) → графики тренировок
- Прогноз ([008](008-ai-inbody-predictor.md)) → пунктирная часть графика InBody

Все цифры детерминированно агрегируются на бэкенде. Специальный кэш не нужен — данных немного на одного пользователя.

---

## 3. User Scenarios

### Scenario 1 — График веса/жира/мышц

**How to test independently**: Создать пользователя с 4+ InBody-замерами, открыть «Аналитика» → «Тело», проверить три графика.

**Acceptance criteria**:

1. **Given** ≥2 InBody-замера.
   **When** пользователь открывает раздел «Тело».
   **Then** видит три графика (вес, % жира, мышечная масса) с осью X = дата, Y = значение; точки на каждом замере, соединённые линией.

2. **Given** включён прогноз ([008](008-ai-inbody-predictor.md)).
   **When** график отображается.
   **Then** правее последней точки показана пунктирная линия прогноза + лента CI.

3. **Given** <2 InBody-замеров.
   **When** пользователь открывает «Тело».
   **Then** видит empty state «Добавьте ещё одно измерение, чтобы увидеть динамику».

4. **Given** пользователь меняет диапазон (1 мес / 3 мес / 6 мес / всё время).
   **When** диапазон применяется.
   **Then** график перерисовывается, агрегаты пересчитываются.

### Scenario 2 — Сравнение двух InBody-замеров

Пользователь хочет сравнить замер «месяц назад» и «сейчас».

**How to test independently**: Открыть раздел сравнения, выбрать два замера, проверить таблицу дельт.

**Acceptance criteria**:

1. **Given** есть ≥2 InBody-замера.
   **When** пользователь открывает «Сравнить замеры», выбирает два.
   **Then** показывается таблица: поле | замер 1 | замер 2 | дельта (абсолютная и %); для дельт вес/жир/мышцы — стрелки и цвета (улучшение/ухудшение в зависимости от цели).

### Scenario 3 — Прогресс по целям

Пользователь хочет видеть, насколько он близок к цели.

**Acceptance criteria**:

1. **Given** пользователь с целью `weight_loss` и стартовым весом 90 кг, целевым 80 кг (target можно указать в профиле).
   **When** он открывает «Прогресс по цели».
   **Then** видит progress bar «6 из 10 кг сброшено», текущее значение, дату старта, ETA на основе прогноза ([008](008-ai-inbody-predictor.md)).

2. **Given** пользователь с целью `muscle_gain` и стартовой мышечной массой 30 кг, целевой 35 кг.
   **When** он открывает «Прогресс».
   **Then** видит аналогичный bar по мышечной массе.

3. **Given** в профиле не указана целевая величина (target weight/muscle).
   **When** открывается раздел.
   **Then** показывается CTA «Укажите цель в профиле».

### Scenario 4 — График объёма тренировок

**Acceptance criteria**:

1. **Given** ≥3 завершённых тренировок.
   **When** пользователь открывает «Тренировки».
   **Then** видит график «тоннаж по неделям» (сумма weight × reps) и «количество тренировок по неделям».

2. **Given** пользователь выбирает конкретное упражнение.
   **When** он открывает «Прогресс по упражнению».
   **Then** видит график «лучший рабочий вес × неделя» и «лучший вес × повторения за подход».

### Scenario 5 — Экспорт PDF-отчёта

**How to test independently**: Нажать «Экспорт PDF», скачать файл, проверить что в нём есть все основные графики.

**Acceptance criteria**:

1. **Given** пользователь на экране аналитики.
   **When** он нажимает «Экспорт PDF» и выбирает диапазон (последние 1/3/6 месяцев).
   **Then** генерируется PDF с разделами: профиль, последние InBody, графики тела, графики тренировок, прогресс по целям; файл скачивается / отправляется на email.

2. **Given** PDF генерируется на сервере.
   **When** пользователь ждёт.
   **Then** показывается прогресс; при готовности — «Скачать» с signed URL (TTL 1 час).

3. **Given** генерация PDF упала.
   **When** пользователь видит ошибку.
   **Then** показывается «Не удалось создать отчёт», кнопка retry; ошибка логируется.

## 4. Functional Requirements

| #      | Requirement                                                                                                |
|--------|------------------------------------------------------------------------------------------------------------|
| REQ-01 | Графики веса, % жира, мышечной массы по InBody-замерам                                                      |
| REQ-02 | Возможность сменить диапазон отображения (1m / 3m / 6m / 1y / all)                                         |
| REQ-03 | Наложение прогноза ([008](008-ai-inbody-predictor.md)) на графике тела (пунктир + CI band)                |
| REQ-04 | Сравнение двух InBody-замеров: таблица дельт по всем полям                                                  |
| REQ-05 | Прогресс по цели (weight_loss/muscle_gain): progress bar + ETA из прогноза                                 |
| REQ-06 | Поле `target_weight_kg` и `target_muscle_kg` в профиле (опц.) для прогресс-бара                            |
| REQ-07 | График общего тоннажа по неделям (сумма weight × reps)                                                     |
| REQ-08 | График количества тренировок по неделям                                                                    |
| REQ-09 | Прогресс по конкретному упражнению (best working weight, best 1-rep max estimate по формуле Epley)         |
| REQ-10 | Экспорт PDF-отчёта по выбранному диапазону                                                                  |
| REQ-11 | PDF включает: профиль (без email/PII вне выбора пользователя), 3 InBody-графика, тоннаж, прогресс по цели  |
| REQ-12 | Опционально отправка PDF на email                                                                           |

## 5. Non-functional Requirements

| #      | Requirement                                                              | Category    |
|--------|--------------------------------------------------------------------------|-------------|
| NFR-01 | Загрузка экрана аналитики ≤1 секунды (на 100 InBody + 100 тренировок)    | Performance |
| NFR-02 | Генерация PDF ≤15 секунд                                                 | Performance |
| NFR-03 | PDF доступен только владельцу (signed URL, TTL 1 час)                    | Security    |
| NFR-04 | Графики работают офлайн на закешированных данных                          | UX          |

## 6. Data Model

Не вводит новых сущностей, читает существующие. Опциональные дополнения в профиле:

**UserProfile (доп. поля)**

| Field             | Type   | Constraints      |
|-------------------|--------|------------------|
| target_weight_kg  | Number | Optional, 30..300 |
| target_muscle_kg  | Number | Optional, 5..120  |
| goal_started_at   | Date   | Optional         |

## 7. Screens

### Screen: Аналитика (главный)

**Elements**:
- Вкладки: «Тело» / «Тренировки» / «Прогресс по цели»
- Селектор диапазона (1m/3m/6m/1y/all)
- Кнопка «Экспорт PDF» в верхнем углу

### Screen: Тело

**Elements**:
- 3 графика: вес, % жира, мышечная масса
- Подпись: текущее значение + дельта от начала диапазона
- Опциональный overlay прогноза

### Screen: Сравнение замеров

**Elements**:
- Два селектора замеров (по дате)
- Таблица дельт

### Screen: Прогресс по цели

**Elements**:
- Progress bar
- Текущее значение / стартовое / целевое
- ETA («Цель достижима к 2026-08-12 при текущем темпе»)
- Если цели нет в профиле → CTA

### Screen: Тренировки

**Elements**:
- График тоннажа по неделям
- График числа тренировок по неделям
- Селектор «Прогресс по упражнению» с автокомплитом

### Screen: Экспорт PDF

**Elements**: модал с выбором диапазона, чекбоксы «включить InBody / тренировки / прогноз», кнопка «Создать PDF».

## 9. API Endpoints

### Серия InBody (для графика)

```
GET /api/v1/analytics/inbody?metric=weight_kg&from=2026-01-01&to=2026-04-28
```
**Response 200**:
```json
{
  "points": [
    { "date": "2026-01-15", "value": 82.0 },
    { "date": "2026-02-15", "value": 80.5 }
  ],
  "forecast": {
    "points": [{ "date": "2026-05-15", "value": 78.0, "ci_low": 77.0, "ci_high": 79.0 }]
  }
}
```

### Сравнение двух замеров

```
GET /api/v1/analytics/inbody/compare?a={uuid}&b={uuid}
```

### Тоннаж и число тренировок

```
GET /api/v1/analytics/workouts?bucket=week&from=...&to=...
```

### Прогресс по упражнению

```
GET /api/v1/analytics/exercise-progress?exercise_id={uuid}&from=...&to=...
```

### Прогресс по цели

```
GET /api/v1/analytics/goal-progress
```
**Response 200**:
```json
{
  "goal": "weight_loss",
  "start_value": 90.0,
  "current_value": 84.0,
  "target_value": 80.0,
  "progress_percent": 60,
  "eta": "2026-08-12",
  "eta_confidence": "medium"
}
```

### Экспорт PDF

```
POST /api/v1/analytics/export-pdf
{ "from": "2026-01-01", "to": "2026-04-28", "sections": ["inbody","workouts","goal"] }
```
**Response 202**: `{"job_id": "..."}` (асинхронно).

```
GET /api/v1/analytics/export-pdf/{job_id}
```
**Response 200**: `{"status":"ready", "url":"<signed>", "expires_at": "..."}`.

## 10. Edge Cases

- 0 или 1 InBody-замер → empty state по графикам тела.
- Очень редкие InBody (1 раз в полгода) → график показывается, но «forecast» помечен low confidence.
- Большая дельта в одну неделю (нереалистичная) → не помечается как ошибка, но в карточке сравнения может быть warning.
- Удаление InBody → агрегаты пересчитываются на лету.
- PDF на 100+ страниц (если очень много истории) → ограничивается 50 страницами; излишек обозначается «...полная история по запросу /export».

---

## 11. Out of Scope

- Дашборды для тренера / организации (B2B)
- Социальное шаринг прогресса
- Продвинутая статистика (1RM с регрессией, periodisation analysis) — фаза 2
- Сравнение с групповой статистикой («ты в топ-15% похудевших среди мужчин 30-40»)
- Экспорт в CSV/Excel — отдельная story

# Dataset-B: inbody_timeseries

**Версия:** 0.1.0
**Spec:** [012-ml-dataset.md](../../../specs/012-ml-dataset.md), Scenario 2
**Используется в:** [008 — AI-прогноз InBody](../../../specs/008-ai-inbody-predictor.md)

---

## Что внутри

Тренировочный набор для time-series регрессии InBody-показателей. Каждая
строка — состояние «условного пользователя» в неделю `t` плюс метка для
прогноза на неделю `t+1`. Один пользователь даёт N строк (`weeks`
аргумент CLI, по умолчанию 8).

**Схема** (см. `dataset_b_inbody_timeseries.csv`):

| Колонка                | Тип    | Описание                                          |
|------------------------|--------|---------------------------------------------------|
| anon_user_id           | string | sha256-HMAC от raw_user_id, 16 hex (REQ-17)        |
| t_week                 | int    | неделя относительно baseline (0..weeks-1)         |
| age                    | int    | возраст из источника                              |
| sex                    | enum   | male / female                                     |
| height_cm              | float  | рост                                              |
| weight_kg              | float  | вес на неделю t (фича)                            |
| body_fat_percent       | float  | % жира на неделю t (фича)                         |
| muscle_mass_kg         | float  | мышечная масса на неделю t (фича; computed из FFM, если в источнике нет) |
| training_volume_t      | float  | тренировок/неделю на момент t                     |
| calories_t             | float  | TDEE-целевые ккал/день (BMR·1.5)                  |
| target_weight_t1       | float  | **label**: вес на неделю t+1                      |
| target_bf_t1           | float  | **label**: % жира на t+1                          |
| target_mm_t1           | float  | **label**: мышечная масса на t+1                  |
| split                  | enum   | train / val / test (70/15/15, без leakage по user)|
| is_synthetic           | bool   | true для всех строк MVP-датасета                  |

## Источники

| ID | Источник | Что используем | Лицензия |
|----|----------|----------------|----------|
| S3 | Kaggle: valakhorasani/gym-members-exercise-dataset | возраст, пол, рост, вес, % жира, частота тренировок, длительность сессии, уровень опыта | См. LICENSES.md |
| S4 | Kaggle: fedesoriano/body-fat-prediction-dataset    | возраст, рост (in), вес (lb), % жира — мужчины          | См. LICENSES.md |

**Реальных лонгитудных InBody-датасетов в открытом доступе нет.** S3/S4 —
кросс-секционные. Поэтому траектории **синтезируются** вокруг каждой
реальной точки (см. `synthesize.py`). `is_synthetic=true` помечает это.

---

## Алгоритм синтеза (главный научный кусок Маши)

Для каждого `Anchor` из источника:

1. Если goal не задан — выводим из BMI (>26 → weight_loss, <21 → muscle_gain, иначе maintenance).
2. Базовая недельная дельта по цели (median):
   - weight_loss: Δw = -0.5 кг, Δfat = -0.25 пп, Δmuscle = +0.05 кг
   - muscle_gain: Δw = +0.25 кг, Δfat = -0.10 пп, Δmuscle = +0.20 кг
   - maintenance: 0/0/0
3. Поправка на частоту тренировок: ±0.05 кг/нед на каждое отклонение от базы 3 раза/неделя.
4. Гауссовский шум σ_w=0.20, σ_f=0.30, σ_m=0.15.
5. Физиологический клип ±1.5 кг/нед, ±1.0 пп/нед, ±0.5 кг/нед — те же,
   что в `app.domain.forecast.predictor`.
6. Все случайности через `random.Random(seed)` — детерминированно.

## Воспроизводимость (REQ-20)

- Фиксированный `seed` в CLI (`--seed`, по умолчанию 42).
- В `dataset_b_inbody_timeseries.meta.json` сохраняется sha256 каждого
  raw-файла. При совпадении хешей и seed — выход bit-for-bit идентичен.
- Никаких сетевых вызовов в pipeline.
- Зависимости — только стандартная библиотека Python.

---

## Privacy (REQ-17, NFR-03)

- Анонимизация: `anon_user_id = HMAC-SHA256(salt, raw_user_id)[:16]`.
  `salt` приходит через CLI и **не коммитится** в репо.
- В этой версии датасета нет дат — весь временной ряд относительный
  (`t_week` от 0). Поэтому offset дат не нужен; функция `date_offset_days`
  оставлена в `anonymize.py` для будущего, когда добавим реальные
  пользовательские траектории.
- Никаких имён, email, точных идентификаторов из источников не сохраняется.

## Известные смещения и ограничения

1. **Синтетика, не реальные данные.** Модель, обученная на этом наборе,
   будет хорошо предсказывать «средние» траектории, но реальная индивидуальная
   вариативность может быть выше. После запуска приложения добавим поток
   реальных траекторий и переобучим.
2. **S4 содержит только мужчин** — это сдвигает баланс датасета. Если в
   обучающей выборке окажется только S4, модель плохо обобщится на женщин.
3. **Нет пропусков и нерегулярности.** Реальные пользователи делают InBody
   нерегулярно; синтетика идёт каждую неделю подряд. Это плюс для обучения
   (чище сигнал), но даёт оптимистичную оценку качества.
4. **Goal выводится по BMI.** В реальных данных пользователь может
   указать другую цель — это создаёт расхождение между training и
   production-распределениями.

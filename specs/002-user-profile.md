# Specification: Профиль пользователя

**Epic:** E1 — Базовая работа пользователя
**User Story:** Пользователь должен указать базовые данные о себе, чтобы AI-блоки могли формировать персонализированный план тренировок и питания.
**Related specs:** [001-auth.md](001-auth.md), [003-inbody-measurements.md](003-inbody-measurements.md), [006-ai-workout-generator.md](006-ai-workout-generator.md), [007-ai-nutrition-planner.md](007-ai-nutrition-planner.md)

---

## 1. User Goal

После подтверждения email пользователь проходит онбординг и заполняет профиль (пол, возраст, рост, вес, цель, уровень подготовки). Эти поля — обязательный вход для AI-генератора тренировок и планировщика питания: без них персонализация невозможна. Пользователь должен иметь возможность отредактировать эти данные позже, когда что-то изменится (например, вес или цель).

---

## 2. Context

Профиль — это статические/полу-статические данные о пользователе. Динамические измерения (вес, % жира, мышечная масса) хранятся в InBody-измерениях ([003](003-inbody-measurements.md)), но в профиле также фиксируется «базовый» вес и рост — на случай, если у пользователя ещё нет ни одного InBody-измерения, AI-блокам нужно с чего-то стартовать.

Когда в InBody появляется свежее измерение, AI-блоки берут вес/жир оттуда, а не из профиля.

---

## 3. User Scenarios

### Scenario 1 — Онбординг профиля после регистрации

После подтверждения email пользователь попадает на пошаговый онбординг и заполняет профиль.

**How to test independently**: Зарегистрироваться, подтвердить email, пройти онбординг до конца, проверить что данные сохранены.

**Acceptance criteria**:

1. **Given** пользователь только что подтвердил email и попал на онбординг.
   **When** он вводит имя, выбирает пол, указывает возраст, рост, вес, цель и уровень подготовки.
   **Then** профиль сохраняется, пользователь попадает на главный экран; AI-генерация теперь доступна.

2. **Given** пользователь не закончил онбординг и закрыл приложение.
   **When** он открывает приложение снова.
   **Then** он попадает на тот же шаг онбординга (промежуточные данные сохранены).

3. **Given** пользователь не заполнил обязательные поля.
   **When** он пытается завершить онбординг.
   **Then** показывается, какие поля незаполнены; завершение блокируется.

### Scenario 2 — Редактирование профиля

Пользователь хочет поменять цель или указать новый вес.

**How to test independently**: Войти, открыть профиль, изменить любое поле, сохранить, перезагрузить — проверить что значение обновилось.

**Acceptance criteria**:

1. **Given** пользователь в настройках профиля.
   **When** он меняет цель (например, с «похудение» на «набор массы») и сохраняет.
   **Then** новое значение сохранено; помечается флаг «план тренировок и питания требует пересборки» (см. [009](009-plan-adaptation-and-chat.md)).

2. **Given** пользователь меняет рост или возраст.
   **When** он сохраняет изменения.
   **Then** обновляются связанные расчёты (BMR, рекомендации по калориям), но текущий план тренировок не пересобирается автоматически (только по триггеру из 009).

### Scenario 3 — Загрузка фото профиля

**How to test independently**: Войти, открыть профиль, загрузить картинку с устройства, проверить отображение.

**Acceptance criteria**:

1. **Given** пользователь в настройках профиля.
   **When** он выбирает изображение из галереи (JPG/PNG, ≤5 MB).
   **Then** изображение загружается, обрезается до 512×512, отображается на главном экране и в шапке.

2. **Given** пользователь выбрал файл >5 MB или неподдерживаемый формат.
   **When** он пытается загрузить.
   **Then** показывается ошибка с конкретной причиной.

### Scenario 4 — Просмотр профиля и удаление аккаунта

**Acceptance criteria**:

1. **Given** залогиненный пользователь.
   **When** он открывает «Мой профиль».
   **Then** видит все свои данные; есть кнопка «Удалить аккаунт» (вызывает flow из [001](001-auth.md)).

## 4. Functional Requirements

| #      | Requirement                                                                                                  |
|--------|--------------------------------------------------------------------------------------------------------------|
| REQ-01 | Онбординг профиля после первого входа: 1 шаг = 1 поле (mobile-first), без возможности пропустить обязательные |
| REQ-02 | Обязательные поля: имя, пол, возраст, рост, вес, цель, уровень подготовки                                     |
| REQ-03 | Поле «цель»: enum `weight_loss` \| `muscle_gain` \| `maintenance`                                            |
| REQ-04 | Поле «уровень подготовки»: enum `beginner` \| `intermediate` \| `advanced`                                   |
| REQ-05 | Сохранение незавершённого онбординга (resume on next session)                                                 |
| REQ-06 | Редактирование любого поля профиля                                                                            |
| REQ-07 | Загрузка фото профиля (JPG/PNG, ≤5 MB), автообрезка до 512×512                                                |
| REQ-08 | Удаление фото профиля                                                                                         |
| REQ-09 | Выбор частоты тренировок 2–6 раз в неделю (нужно для AI-генератора, см. [006](006-ai-workout-generator.md))   |
| REQ-10 | Поле «аллергии/непереносимости» (множественный выбор + custom text), нужно для [007](007-ai-nutrition-planner.md) |
| REQ-11 | При смене цели или частоты — установка флага «план требует пересборки» (см. [009](009-plan-adaptation-and-chat.md)) |
| REQ-12 | Расчёт и хранение BMR (формула Mifflin-St Jeor) при каждом изменении веса/роста/возраста/пола                 |

## 6. Data Model

**Entity: UserProfile**

| Field                     | Type      | Constraints                                                 | Description                                |
|---------------------------|-----------|-------------------------------------------------------------|--------------------------------------------|
| user_id                   | UUID      | PK + FK → User, required                                    | 1:1 с аккаунтом                            |
| name                      | String    | Required, 1..50 chars                                       | Имя для отображения                        |
| sex                       | Enum      | Required, `male` \| `female`                                | Биологический пол (для BMR и AI-моделей)   |
| birth_date                | Date      | Required, прошлая дата, возраст 14..100                     | Хранится дата, возраст вычисляется         |
| height_cm                 | Number    | Required, 100..250                                          |                                            |
| baseline_weight_kg        | Number    | Required, 30..300                                           | Стартовый вес; актуальный — из InBody       |
| goal                      | Enum      | Required, `weight_loss` \| `muscle_gain` \| `maintenance`   |                                            |
| training_level            | Enum      | Required, `beginner` \| `intermediate` \| `advanced`        |                                            |
| training_frequency        | Integer   | Required, 2..6                                              | Раз в неделю                               |
| allergies                 | String[]  | Optional                                                    | Enum предустановленных + custom строки     |
| photo_url                 | String    | Optional                                                    | Путь в storage                             |
| bmr_kcal                  | Number    | Computed, immutable извне                                   | Mifflin-St Jeor, пересчитывается           |
| onboarding_completed_at   | Timestamp | Optional                                                    | Null = онбординг не закончен               |
| plan_rebuild_required     | Boolean   | Default false                                               | Флаг для AI-блоков                         |
| updated_at                | Timestamp | Required                                                    |                                            |

## 7. Screens

### Screen: Онбординг (пошаговый)

**Purpose**: Собрать обязательные поля профиля одним проходом.

**Elements** (по шагам):
1. Имя
2. Пол (radio)
3. Дата рождения (date picker)
4. Рост (slider/input cm)
5. Вес (slider/input kg)
6. Цель (3 карточки)
7. Уровень подготовки (3 карточки с описанием)
8. Частота тренировок 2–6 раз/нед (slider)
9. Аллергии (multi-select + custom)
10. Фото (опционально, можно пропустить)

**States**:
- Linear progress indicator сверху
- На каждом шаге кнопки «Назад» / «Далее»

**Navigation**: Email confirmed → Онбординг → Главный экран.

### Screen: Мой профиль

**Purpose**: Просмотр и редактирование профиля.

**Elements**:
- Аватар + имя
- Все поля из data model в виде списка с edit-иконкой
- Секция «Аккаунт»: email, кнопки «Сменить пароль», «Удалить аккаунт», «Скачать мои данные»

**States**:
- Loading, Error
- Edit mode: inline-редактирование одного поля

**Navigation**: Главный экран → Профиль; Профиль → Auth flows для смены пароля/удаления.

## 9. API Endpoints

### Получить профиль

```
GET /api/v1/profile
```

**Response 200**: все поля UserProfile + `email`.

### Обновить профиль (PATCH)

```
PATCH /api/v1/profile
```

**Request** (любое подмножество полей):
```json
{ "goal": "muscle_gain", "training_frequency": 4 }
```

**Response 200**: обновлённый профиль.

### Загрузить фото

```
POST /api/v1/profile/photo    (multipart/form-data, поле "file")
```

**Errors**:
| Code | Situation              |
|------|------------------------|
| 413  | Файл >5 MB             |
| 415  | Неподдерживаемый формат |

### Удалить фото

```
DELETE /api/v1/profile/photo
```

### Завершить онбординг

```
POST /api/v1/profile/complete-onboarding
```

Срабатывает только если все обязательные поля заполнены, иначе 400 `incomplete_profile` со списком пустых полей.

### Экспорт данных (GDPR)

```
GET /api/v1/profile/export
```

**Response 200**: JSON со всеми пользовательскими данными (профиль, InBody, тренировки, планы).

## 10. Edge Cases

- Возраст <14 → блокировка регистрации (несовершеннолетние требуют согласия родителей, в MVP не поддерживаем).
- Рост <100 или >250 → ошибка валидации.
- Все аллергии сняты → пустой массив, не null.
- Пользователь не завершил онбординг → доступ к AI-функциям закрыт, главный экран показывает CTA «Завершить настройку».

---

## 11. Out of Scope

- Социальные функции (друзья, лента) — не входит в ТЗ
- Несколько профилей на один аккаунт (для тренеров) — отдельная B2B-история
- Смена единиц измерения (фунты/футы) — only metric в MVP
- Импорт профиля из Apple Health / Google Fit
